Q1:Discuss the scope of var, let, and const.

In JavaScript, the scope of variables varies depending on how they are declared. Variables declared with var have function scope, which means they are accessible throughout the entire function where they are declared. However, they are also hoisted, so they are moved to the top of their function or global scope during compilation, potentially leading to unexpected behavior. On the other hand, both let and const have block scope, meaning they are confined to the block (e.g., if statement or loop) in which they are defined. This makes them more predictable and safer to use, especially within modern JavaScript. Additionally, let and const are hoisted but not initialized until they are declared in the code, ensuring that they can only be accessed after declaration. const specifically is used for variables that should not be reassigned, while let allows reassignment as needed.

Q2: Tell us the use cases of null and undefined.

Null is used to represent the intentional absence of any object value, often as a programmer's choice. It's a value you can assign to a variable to signify that it has no valid object or value at a given moment. undefined, on the other hand, typically indicates a variable that has been declared but hasn't been assigned any value yet or a missing function argument. It's usually an unintentional absence of a value.

Q3: What do you mean by REST API?

A REST API (Representational State Transfer Application Programming Interface) is a set of rules for building web services. It uses standard HTTP methods (like GET, POST, PUT, DELETE) to allow communication and data exchange between clients (e.g., web or mobile apps) and servers via URLs. REST APIs are known for their simplicity and scalability, making them widely used in web development for data retrieval and manipulation.
